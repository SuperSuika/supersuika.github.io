云服务器Linux系统环境下，执行python程序killed问题解决方法。

2020-11-16

服务器操作系统：CentOS 7.6 64bit
Python：3.6.x
在云服务器部署了python程序后运行，直接被killed。

1、输入free -m 查看swap分区情况
2、这里应该显示swap分区为0，表示配有分配，接下来开始执行
cd/opt 切换到我的opt目录
mkdir swap 创建文件夹
cd swap 切换到swap文件夹下
3、使用dd命令新建分区文件，这里分配1G
dd if=/dev/zero of=swapfile bs=1M count=1024
4、将swapfile增加为swap分区，并开启
mkswap swapfile
swapon swapfile
5、再次free -m swap分区已经变大
